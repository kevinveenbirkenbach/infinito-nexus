# Documentation Lint 📄

Markdown-link resolvability, CLI-category README presence, and `@todo` transparency.

Tests in this directory MUST only cover documentation artefacts (`*.md`, README presence) and inline TODO markers. Code-level rules MUST live under [`python/`](../python/), [`shell/`](../shell/), or [`yaml/`](../yaml/).

For framework and `make test-lint` usage see [lint.md](../../../../docs/contributing/actions/testing/lint.md).
