# Review Requirements 🔍

You MUST open a Pull Request only when it is ready for real review.

The matching Pull Request template is the SPOT for PR structure, required author input, and review context. You MUST use the guidance in [Pull Request Templates](pull-request.md) and orient your Pull Request to it.

The [lint.md](testing/lint.md) page is the SPOT for repository-wide code quality expectations such as diff quality, maintainability, and conventions.

In addition to the templates, keep these repository-wide review rules in mind:

- You MUST keep commit history clear and reviewable.
- Code ownership is defined in [CODEOWNERS](../../../.github/CODEOWNERS).
- Auth, proxy, TLS, networking, migrations, and shared abstractions MUST receive extra review attention.
- You SHOULD attach screenshots, traces, logs, and migration notes when they materially help validate the change.

More information [here](https://s.infinito.nexus/reviewguide).
