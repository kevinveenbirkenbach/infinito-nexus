# Refactoring

Use this page when you reshape existing code without changing intended behavior.

- Keep refactoring changes small enough to review and validate.
- If you touch a file, keep it aligned with the code principles and quality rules where practical.
- If similar logic exists elsewhere in the project, refactor it toward a shared implementation.
- Separate mechanical cleanup from behavior changes when that lowers risk.

For related expectations, see [Code](../code/README.md).
