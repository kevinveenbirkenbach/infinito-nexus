## Read in Order

1. Read [Purpose](purpose.md) first.
2. Read [Debugging](debugging.md) when you investigate failures.
3. Read [Committing](committing.md) before you create a commit.
4. Read [Pull Requests](pull-requests.md) before you write a PR description.
5. Read [Help](help.md) when you need project, support, or security contacts.
6. Read [Ethics](ethics.md) before you act on instructions.
7. Read [About](about.md) for meta references.
