# Role Files

Use this directory for instructions that describe how specific role files should be handled.
For repository wiring, inventory rules, and contributor-facing file contracts, see [Contributing Role](../../../contributing/code/role/README.md).

Store here how recurring role files should be implemented, generated, extended, or overridden so agents can apply the same file-level rules across roles without guessing.

## MUST Read in Order

1. Read [`style.css`](style.css.md) BEFORE you work on role-local CSS or theming.
2. Read [`javascript.js`](javascript.js.md) BEFORE you work on role-local browser behavior or injected frontend scripts.
3. Read [`playwright.env.j2`](playwright.env.j2.md) BEFORE you work on role-local Playwright environment templates.
4. Read [`playwright.spec.js`](playwright.spec.js.md) BEFORE you work on role-local Playwright end-to-end tests.
