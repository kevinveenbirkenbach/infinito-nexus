# Log File Inspection

Use this page when the operator has manually placed `*.log` or `*job-logs.txt` files in the repository workdir for ad-hoc inspection. For the fully automated CI triage that fetches logs itself via `gh`, see [pipeline.md](pipeline.md).

- You MUST inspect the relevant logs in the provided `*job-logs.txt` or `*.log` files.
- You MUST treat the provided log files as the source of truth for the failure they describe.
